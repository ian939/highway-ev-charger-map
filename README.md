# 고속도로 휴게소 전기차충전소 현황 지도

엑셀 현황표를 Kakao 지도에 시각화한 단일 HTML. 운영사별 버튼으로 설치 분포를 필터(OR)한다.

## 결과물
- `index.html` — 바로 열어 보는 지도 페이지 (좌표 데이터 인라인 임베드, 추가 빌드 불필요)

## 데이터 요약
- 대상: 메인 섹션 휴게소 중 운영사 데이터 보유 **236곳**, 총 충전기 **2,261대**
- 운영사 17개 컬럼(환경부·한전·기타 4종·SK 차수별)을 그룹 버튼으로 토글
- 건설중·미확정 부지 4곳은 읍·면 단위 근사 좌표(`근사` 표시)

## 휴게소 추가 (CSV 업로드, 브라우저에서)
지도 좌측 패널에서.
1. **① 샘플 양식 다운로드** → `휴게소_충전소_추가양식.csv` 받기 (엑셀과 동일한 컬럼)
2. 엑셀/메모장에서 휴게소명·시설구분·노선명·주소 + 운영사별 충전기 수를 채움
   - **새 사업자/사업**을 넣으려면 양식 맨 뒤에 컬럼을 추가하고 그 칸에 값만 채우면 됨.
     기존 17개 컬럼은 그대로 둔다. 헤더의 기본 4칸(휴게소명·시설구분·노선명·주소)을
     뺀 나머지는 전부 사업자 컬럼으로 인식되어, 새 사업자는 색상·필터 버튼이
     "추가 사업자" 그룹에 자동 생성된다.
3. **② CSV 업로드** → 주소를 클라이언트에서 지오코딩(Kakao services)해 지도에 즉시 추가

> 업로드분은 **현재 화면에만** 반영되고 새로고침하면 사라진다. 영구 반영은
> 원본 엑셀에 추가 후 아래 빌드를 다시 돌린다.

## 다시 빌드하려면 (엑셀이 갱신됐을 때)
```bash
pip install openpyxl requests python-dotenv   # 최초 1회
python build_geocode.py    # 엑셀 → Kakao 지오코딩 → data.json
python build_html.py       # data.json + JS키 → index.html 생성
```

## Kakao 키 (.env)
`.env`에 두 키를 둔다. `.env`는 절대 커밋 금지(.gitignore 처리됨).
```
KAKAO_REST_KEY=...   # 지오코딩용. HTML에 포함되지 않음
KAKAO_JS_KEY=...     # 지도 표시용. index.html에 주입됨(클라이언트 노출 불가피)
```

## ⚠️ 도메인 등록 필수 (지도가 안 뜰 때 첫 번째 확인)
Kakao JavaScript 키는 **등록된 웹 도메인에서만** 동작한다. 미등록 시 SDK가 401
(`domain mismatched!`)을 반환하고 지도가 안 뜬다.

Kakao Developers → 내 애플리케이션 → 앱 설정 → 플랫폼 → **Web → 사이트 도메인**에
열어 볼 주소를 등록한다.
- 로컬: `http://localhost:8765` (`python -m http.server 8765`로 띄울 때)
- 배포: 실제 배포 도메인

> `file://`로 바로 열면 도메인 검증을 통과하지 못한다. 로컬 확인은 위처럼
> 간단한 http 서버로 띄워서 본다.
